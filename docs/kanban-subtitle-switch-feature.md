# 看板字幕切换功能文档

## 功能概述

字幕切换功能允许一条字幕在不同时间点显示不同的文字内容，实现动态文字更新效果。

## 核心功能

### 1. 切换开关

- 位于字幕配置面板底部
- 绿色主题切换按钮
- 开启后展开切换文字列表

### 2. 切换文字管理

#### 添加切换文字
- 点击"+ 添加切换文字"按钮
- 自动获取当前视频时间作为出现时间
- 默认文字内容为"新文字"

#### 编辑切换文字
- **文字内容**: 输入要显示的文字
- **出现时间**: 设置文字出现的时间点（秒）
  - 手动输入数值
  - 点击"当前"按钮获取播放器时间

#### 删除切换文字
- 点击切换项右上角的"删除"按钮
- 立即从列表中移除

### 3. 显示逻辑

#### 文字优先级
```
当前时间: 5.5 秒

切换列表:
├─ 文字1: 2.0秒 "开始"
├─ 文字2: 4.0秒 "继续"  ✓ 最新的已到达时间点
└─ 文字3: 7.0秒 "结束"  (未到达)

显示: "继续"
```

#### 选择规则
1. 筛选所有已到达时间点的切换项
2. 按时间降序排列（最新的在前）
3. 选择第一个（最新的）切换文字
4. 如果没有到达任何时间点，显示原始文字

#### 时间轴示例
```
时间 →
│
├─ 0.0s: 原始文字 "字幕"
│
├─ 2.0s: 切换到 "开始"
│         ↓ 文字变化
│
├─ 4.0s: 切换到 "继续"
│         ↓ 文字变化
│
├─ 7.0s: 切换到 "结束"
│         ↓ 文字变化
│
└─ 退出时间
```

## 技术实现

### 数据结构

```typescript
// 切换配置
interface KanbanSubtitleSwitch {
  enabled: boolean;               // 是否启用切换
  items: KanbanSubtitleSwitchItem[]; // 切换文字列表
}

// 单条切换项
interface KanbanSubtitleSwitchItem {
  id: string;    // 唯一标识
  text: string;  // 文字内容
  time: number;  // 出现时间（秒）
}
```

### 渲染逻辑

```typescript
// 计算当前显示文字
let displayText = subtitle.text; // 默认使用原始文字

if (subtitle.switch.enabled && subtitle.switch.items.length > 0) {
  // 筛选已到达时间点的切换项
  const validItems = subtitle.switch.items
    .filter((item) => currentTime >= item.time)
    .sort((a, b) => b.time - a.time); // 降序排列

  // 使用最新的切换文字
  if (validItems.length > 0) {
    displayText = validItems[0].text;
  }
}

// 渲染文字
return <div>{displayText}</div>;
```

### 数据流

```
用户添加切换文字
  ↓
SubtitleConfigPanel
  ↓
onUpdate({ switch: { items: [...] } })
  ↓
KanbanConfigEditor
  ↓
config.subtitles[i].switch
  ↓
SubtitleRenderer
  ↓
根据 currentTime 选择文字
  ↓
渲染到屏幕
```

## 与其他功能的配合

### 1. 与进入动画配合

- 原始文字使用进入动画
- 切换时文字直接替换（无动画）
- 保持字幕位置和样式

```typescript
// 进入时带动画
0.0s: "原始文字" (slam-bounce 动画)

// 切换时无动画
2.0s: "新文字" (直接替换)
```

### 2. 与缩放配合

- 缩放动画仍然生效
- 切换仅改变文字内容
- 位置和大小由缩放控制

```typescript
0.0s: "原始" (位置A, 48px)
1.0s: 开始缩放 → (位置B, 72px)
2.0s: "新文字" (位置B, 72px) ← 文字切换
```

### 3. 与退出配合

- 退出时使用当前显示的文字
- 淡出动画正常执行

```typescript
9.5s: "最后文字"
10.0s: 淡出 "最后文字"
```

## 注意事项

### 1. 时间点顺序

- 切换项无需按时间排序
- 系统自动计算最新时间点
- 建议按时间顺序添加以便管理

### 2. 时间点冲突

如果多个切换项时间相同：
```typescript
items: [
  { time: 2.0, text: "A" },
  { time: 2.0, text: "B" }, // 相同时间
]
```
系统会选择数组中靠后的项（后添加的优先）

### 3. 空列表

- 切换功能开启但列表为空
- 显示原始文字
- 提示用户添加切换项

### 4. 时间范围

- 切换时间应在字幕显示范围内
- `enterTime <= switchTime < exitTime`
- 超出范围的切换项不会生效

### 5. 文字长度

- 不同切换文字长度可能不同
- 建议测试最长文字的显示效果
- 必要时调整字体大小或位置

## 性能优化

### 1. 筛选优化

```typescript
// 高效的时间点筛选
const validItems = items
  .filter(item => currentTime >= item.time) // 过滤
  .sort((a, b) => b.time - a.time)[0];     // 排序取第一个
```

### 2. 渲染优化

- 仅在文字变化时重新渲染
- 保持其他属性（位置、大小）不变
- 无需额外的动画计算

### 3. 内存优化

- 切换项存储在配置中
- 随工作流保存到 localStorage
- 不占用额外内存

## 常见问题

### Q: 切换时有闪烁？
A: 
- 切换是瞬时的，无过渡动画
- 如需平滑过渡，考虑使用多条字幕
- 或等待后续淡入淡出效果更新

### Q: 文字没有切换？
A: 检查：
1. 切换功能是否开启
2. 当前时间是否到达切换点
3. 切换列表是否为空
4. 时间点是否在字幕显示范围内

### Q: 如何实现倒计时？
A: 
```typescript
原始文字: "准备"
切换项:
  - 时间 3.0s, 文字 "3"
  - 时间 4.0s, 文字 "2"
  - 时间 5.0s, 文字 "1"
  - 时间 6.0s, 文字 "开始！"
```

### Q: 可以切换回原始文字吗？
A: 
- 可以添加一个切换项，文字设为原始文字
- 或者不添加该时间段的切换项
- 原始文字始终是备选项

### Q: 切换会影响其他动画吗？
A: 
- 不影响进入动画
- 不影响缩放动画
- 不影响退出动画
- 仅改变显示的文字内容
